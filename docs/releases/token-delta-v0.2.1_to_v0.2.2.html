Adopted Improvements and Regressions

Improvements Adopted vs v0.2.1

  • Local poetry-run pre-commit hooks: version drift between local and CI is now structurally impossible. Expected to eliminate the recurring CI lint/format failure category entirely from v0.3.x onward.
  • Binary smoke test (14 scenarios): broken binary class of problems cannot pass CI again.
  • Token report CI gate: being added this session — G9 skip will be structurally blocked going forward.
  • No session boundary cost: continued session avoided ~20,000–30,000 tokens of context reload overhead.

Regressions / New Friction

  • Rework ratio increased (14% → 24%): smaller scope amplified the relative cost of fixed CI overhead. Not a real regression — absolute rework cost was flat; ratio is a scope artifact.
  • G9 Token Analytics skipped again (2nd release in a row): now being structurally enforced via CI gate.
  • Pre-commit version drift added one extra CI cycle: the intermediate pin-update fix was a symptom fix, not the root cause. Root cause resolved one commit later.

Efficiency Verdict and Forecast

Verdict: IMPROVED (absolute) / REGRESSED (ratio)

Total token spend dropped significantly due to smaller scope. However the rework ratio worsened because the same CI overhead applied to a much smaller productive base. The structural fixes in this release (local hooks, smoke test, CI enforcement) are expected to reduce rework ratio below 10% for v0.3.x.
