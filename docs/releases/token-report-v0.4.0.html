Top Token Sinks

  1. TDD build — new parsers and tests (~50% of total)
    The Tag-Value parser required a full custom line-by-line parser (not just a wrapper around existing logic). The YAML parser is a thin wrapper but required understanding how to correctly override the format field in the returned NormalizedSBOM. Together with 78 new tests covering happy paths, edge cases, and error paths, this is the justified dominant cost for a two-format feature addition.
  2. mypy type annotation correction loop (~4% of total)
    json.loads() returns Any; explicit typed variables needed to satisfy mypy strict mode in _load_raw_doc. Also required types-PyYAML stubs to be installed. Two correction cycles: one lint (unused import), one type (3 mypy errors). Total rework cost was modest.
  3. ADR-009 authoring (~8% of total)
    First new ADR since v0.2.0. Required designing five architectural sub-decisions (detection priority, constants naming, schema policies for two formats, parser factoring, YAML library). Time well spent given the decision complexity.
  4. Documentation sync (~7% of total)
    Adding the new SPDX serialization rows to the README and user-guide, plus updating the agent-briefing.md module map with three new function signatures. Higher than v0.3.0 because the formats table schema is more complex (added schema policy column).
  5. Token analytics reports (~6% of total)
    Consistent per-release overhead. Pre-tag execution enforced by CI — no post-tag rework.
