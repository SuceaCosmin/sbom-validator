5 — Top Workflow Gaps

MINOR
G9 and G10 remain pending until agent execution — tag cannot be pushed until complete
While the CI gate correctly enforces pre-tag execution of G9 and G10, there is a small sequencing window where the release is notionally "ready" (G1–G8 all pass) but the tag cannot be pushed. This is the intended design, not a gap, but it requires the orchestrator to proactively schedule the token-analyst and workflow-analyst agents before the human final approval checkpoint.
Root cause: G9 and G10 require the completed release cycle as input — they cannot run before G8.
Fix: Already handled by the CI gate. No structural change needed. Document the expected sequencing explicitly in the release brief template.
MINOR
xmlschema supply-chain risk remains as MEDIUM residual — no automated CVE scanning in CI
The G5 security verdict accepted a MEDIUM residual risk: xmlschema and elementpath are not pinned to exact versions. A future CVE in these packages would not be automatically surfaced until a developer upgrades and notices, or until G5 is re-run. No pip-audit or safety step exists in CI.
Root cause: Project maturity — dependency pinning and CVE scanning are not yet part of the CI pipeline.
Fix: Add pip-audit as an optional CI check in a future cycle. Low effort, addresses the residual MEDIUM finding from G5.
MINOR
Reviewer findings not explicitly documented as a distinct artifact
G4 passed with clean lint/type output as the primary evidence. No separate reviewer findings summary artifact was produced (recommended in the v0.2.2 gap report). The operating model states "no open CRITICAL/MAJOR (or approved deferrals)" as the exit criterion — a clean verdict document would make the compliance evidence more explicit.
Root cause: No template or convention for a reviewer findings summary exists yet.
Fix: Low-effort — add a short "Reviewer Findings" section to the TASKS tracker template so G4 evidence is captured in a structured way.
WATCH
Python 3.12 not included in test matrix this cycle
v0.2.2 ran tests on Python 3.11 and 3.12. v0.3.0 only reports 3.11. (inferred — TASKS tracker G3 evidence references Python 3.11 only) If CI dropped 3.12 coverage implicitly, a future regression on 3.12 may go undetected.
Root cause: Unknown — may be a CI matrix simplification or an oversight in the G3 evidence record.
Fix: Verify CI matrix includes Python 3.12. If it was dropped intentionally, document the decision in the TASKS tracker.
