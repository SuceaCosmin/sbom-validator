Adopted Improvements and Regressions

Improvements Adopted vs v0.2.2

  • CI lint/format cycles eliminated (−4 cycles): Local poetry-run pre-commit hooks fully effective. This was the single largest token sink in v0.2.2 (~25% of total). Confirmed zero cycles in v0.3.0.
  • G5 Security gate executed for first time (D1 resolved): D1 had been deferred across two prior releases. Formal review conducted, path traversal risk bounded, verdict APPROVED. Accumulated risk closed.
  • G9 Token Analytics executed pre-tag: CI enforcement gate from v0.2.2 worked as intended. This report is being generated before the v0.3.0 tag — first time in the project. Prevents post-tag report rework.
  • Feature branch PR workflow: v0.3.0 used a proper feature branch (feature/cyclonedx-multi-version) with PR merge, rather than committing directly to develop. Cleaner history; enables independent review.
  • Zero rework on first CI push: G4 and G6 both passed on first attempt. Structural tooling improvements compounded — no correction loops needed.

Regressions / New Friction

  • ADR-001 context section is stale: References CDX 1.6 only; now describes a narrower truth than the implementation. Low severity — the decision itself is unchanged. Should be updated in v0.4.x planning to avoid repeated G2 staleness notes.
  • Security review as one-time catch-up (not recurring): D1 was resolved but no recurring security review cadence was established. Risk: future dependency additions accumulate without review. Mitigation: trigger G5 on each new dependency, not just each release.
  • Fixture file proliferation: 6 new fixture SBOMs (one per CDX version x format). No generator script exists. Adding CDX 1.7 would require manual fixture creation again.

Efficiency Verdict and Forecast

Verdict: IMPROVED

v0.3.0 delivered a new user-facing feature (CycloneDX multi-version support) and resolved a deferred security item at a lower total token cost than v0.2.2's pure bug-fix/hardening scope. The structural improvements shipped in v0.2.2 (local hooks, smoke test, G9 CI enforcement) paid compound dividends this release — CI overhead dropped from the dominant cost category to near-zero. The rework ratio improved from ~24% to ~6–9%.
