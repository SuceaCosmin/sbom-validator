4. Top Workflow Wins

  1. Pre-commit hooks (ruff + ruff-format) eliminated all remote CI lint failures for the third consecutive release.
    Developer commits go through local formatting and lint checks automatically. This structural change from v0.2.2 continues to be the single highest-ROI improvement in the project's workflow history. In v0.4.0: zero CI format/lint cycles, pre-commit caught 1 lint issue + 6 format issues at commit time rather than in remote CI.
  2. ADR-009 provided clear interface contracts before implementation started.
    The Python stubs in ADR-009 (detect_format return type, _parse_spdx_document signature, parse_spdx_yaml, parse_spdx_tv) gave the Developer unambiguous contracts to implement against. The TV-before-YAML detection priority decision in the ADR prevented an implementation ambiguity that would have required a correction loop if left to the Developer to resolve independently.
  3. TDD discipline maintained: 78 tests written before implementation.
    All tests were written (and passed in the expected "fail" state) before any implementation code was written. The test suite caught the SPDXID-missing edge case in the TV parser, the YAML array-at-root error path, and detection priority ordering — all of which would have been harder to verify post-implementation.
  4. Single-session execution with full gate coverage.
    The entire release cycle from intake to workflow report was completed in one session, maintaining cache warmth throughout and avoiding multi-session context reload overhead. All 11 gates executed in sequence without interruption.

5. Top Workflow Gaps

  1. mypy strict mode + json.loads/yaml.safe_load return type: recurring pattern requiring correction loop.
    Root cause: Both json.loads() and yaml.safe_load() return Any in their type stubs. Functions with explicit return type annotations (like _load_raw_doc) require typed intermediate variables. This pattern recurs whenever a new format or parser is added. In v0.4.0: 3 mypy errors, 1 correction loop.
    Impact: Low (small correction loop), but predictable and avoidable.
    Fix: Add a "typed intermediate variable" pattern note to developer agent definition. Developer should proactively use typed intermediates when calling Any-returning functions.
  2. ADR-001 main document not updated to reflect new detection priority (stale prose).
    Root cause: The agent-briefing.md ADR table was updated (correct), but the full ADR-001 document prose still describes the old two-path detection (JSON → CycloneDX XML). ADR-009 defines the new priority without cross-referencing ADR-001.
    Impact: Low for current release (briefing is up to date). Medium risk for future agents who read ADR-001 directly.
    Fix: Add an "ADR cross-reference check" to the G7 Docs Sync checklist: when a new ADR extends or modifies a previous ADR, the previous ADR's prose must be updated to reference the new ADR.
  3. Fixture file count growth: 8 new files manually authored without a generation helper.
    Root cause: v0.4.0 added 4 TV + 4 YAML fixtures, all manually adapted from existing JSON fixtures. No automated fixture generator exists.
    Impact: Low for v0.4.0 (8 small files). Medium at v0.5.0+ if more formats are added.
    Fix: Consider a fixture generation helper script for v0.5.0 if additional SPDX formats (e.g., RDF) are planned.
