New Friction Points Identified in v0.4.0

  • mypy strict mode + json.loads/yaml.safe_load return type: Both functions return Any. Requires explicit typed intermediate variables to satisfy --strict. Two correction cycles needed (3 errors). Low cost individually but a recurring pattern as new parsers are added.
  • Fixture file count growth: v0.4.0 added 8 fixture files (4 TV + 4 YAML). Each new serialization format doubles the fixture set. Without a generation script, this will become a manual maintenance burden at v0.5.0+.
  • agent-briefing.md becoming stale between releases: The Module Map section required updates to three function signatures and the NormalizedSBOM.format field type. A per-release "briefing freshness check" step in the planning gate would catch this earlier.

Net Efficiency Verdict

FLAT (scope-adjusted)

Raw token spend increased ~55–60% vs v0.3.0, but this is proportional to the ~1.5x increase in implementation surface (2 new parsers vs 0, 78 new tests vs 15, new ADR vs no ADR). Rework ratio improved slightly (4–6% vs 6–9%). The absolute rework token cost held flat despite larger scope — a positive signal that the structural CI improvements (pre-commit hooks, mypy-first development) are mature and consistently effective.

The per-token efficiency (quality outcomes per token spent) is approximately flat vs v0.3.0. No significant regression or improvement in process efficiency; the increase in raw tokens is justified by feature scope.
